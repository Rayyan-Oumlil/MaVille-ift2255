Use Case: Signaler un problème
Actor: Citoyen
Preconditions: L’utilisateur est connecté
Main Flow:
1. L’utilisateur clique sur "Signaler un problème"
2. Le système affiche un formulaire
3. L’utilisateur remplit les champs
4. Le système vérifie les données
5. Si valide, enregistre le signalement et notifie un gestionnaire
Alternate Flow:
- Si les données sont invalides, un message d’erreur est affiché
Postconditions: Le signalement est enregistré de manière persistante
